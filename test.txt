user registration flow

1. User fills signup form → frontend validates
   ↓
2. Frontend sends POST /api/auth/signup
   ↓
3. Backend validates data
   ↓
4. Backend hashes password with bcrypt
   ↓
5. Backend saves user to database
   ↓
6. Backend generates JWT token
   ↓
7. Backend responds with token + user data
   ↓
8. Frontend stores token in localStorage
   ↓
9. Frontend updates AuthContext (user logged in)
   ↓
10. Frontend redirects to dashboard


login flow


1. User enters credentials
   ↓
2. Frontend sends POST /api/auth/login
   ↓
3. Backend finds user by email
   ↓
4. Backend compares password hash
   ↓
5. Backend generates JWT token
   ↓
6. Backend responds with token + user data
   ↓
7. Frontend stores token in localStorage
   ↓
8. Frontend updates AuthContext
   ↓
9. Frontend redirects to dashboard


crud operations 

1. User performs action (e.g., create task)
   ↓
2. Frontend sends POST /api/tasks with JWT in header
   ↓
3. Backend auth middleware intercepts request
   ↓
4. Middleware extracts JWT from Authorization header
   ↓
5. Middleware verifies JWT signature
   ↓
6. Middleware decodes userId from token
   ↓
7. Middleware attaches user to req.user
   ↓
8. Request proceeds to controller
   ↓
9. Controller creates task associated with user
   ↓
10. Backend responds with created task
   ↓
11. Frontend updates UI


logout flow

1. User clicks logout
   ↓
2. Frontend removes token from localStorage
   ↓
3. Frontend clears AuthContext state
   ↓
4. Frontend redirects to login page
   ↓
5. Subsequent requests fail (no token) → auto-redirect


Required Tools & Technologies
Backend Stack
Core:

Node.js (v18+) - Runtime environment

Express.js - Web framework

MongoDB + Mongoose - Database (or PostgreSQL + Sequelize)

Authentication:

jsonwebtoken - JWT generation/verification

bcryptjs - Password hashing

Validation & Security:

express-validator or Joi - Input validation

helmet - Security headers

cors - Cross-origin resource sharing

express-rate-limit - Rate limiting

Development:

nodemon - Auto-restart server

dotenv - Environment variables

morgan - Request logging

Frontend Stack
Core:

React (v18+) - UI library

Vite - Build tool (faster than CRA)

React Router DOM - Routing

HTTP Client:

Axios - API requests

Styling:

Tailwind CSS - Utility-first CSS (recommended)

Or Material-UI / Bootstrap

State Management:

Context API (built-in) - Good enough for this project

Or Zustand / Redux Toolkit for complex apps

Form Handling:

React Hook Form - Form validation

Yup or Zod - Schema validation

Development:

ESLint - Code linting

Prettier - Code formatting
